DevSecBox is a comprehensive workflow orchestration extension that transforms security testing by providing a visual, drag-and-drop interface for building complex automation chains. It functions as a command-line adapter, enabling execution of shell scripts, PowerShell commands, and external programs directly from HTTP traffic analysis. The extension creates dynamic workflows through interconnected components called Triggers, Catchers, Senders, Spoofers, and Solvers, allowing security professionals to automate everything from simple data transformations to multi-step vulnerability discovery processes.

Features

  • Visual workflow designer with drag-and-drop components (Triggers, Catchers, Senders, Spoofers, Solvers)
  • Cross-platform command execution supporting shell, PowerShell, and compiled programs
  • Live and manual execution modes with configurable concurrency and timeout controls
  • HTTP request/response manipulation with data spoofing capabilities
  • Comprehensive JSON processing including validation, beautification, and transformation
  • Automated vulnerability reporting through structured JSON issue generation
  • Interactive HTTP request builder and testing functionality
  • Configurable thread pools and semaphore-based process management
  • Built-in search, replace, and export capabilities for workflow data
  • Real-time audit integration with customizable severity and confidence levels

Usage

  1. Create a workflow by right-clicking in the DevSecBox panel and selecting Trigger to establish the entry point for your automation chain.
  2. Add Catcher components to process incoming HTTP data, using the colored placeholder buttons to insert request/response fields into your shell commands or scripts.
  3. Configure processing logic in each Catcher's in tab using shell syntax, PowerShell commands, or JSON transformations, with data flowing between connected components through out tabs.
  4. Use the Sender component to make custom HTTP requests based on processed data, and the Spoofer component to replace specific content in responses.
  5. Add a Solver component in Live mode to automatically generate structured vulnerability reports from your workflow output using predefined JSON formats.
  6. Test workflows manually by right-clicking requests in Proxy history and selecting Send to DevSecBox, or enable Live Workflow mode for automatic processing.
  7. Configure concurrency settings, timeouts, and thread limits using the booster tab to optimize performance for your specific use case.
  8. Use the context menu options for JSON import/export, search/replace functionality, and workflow testing with mock data.
