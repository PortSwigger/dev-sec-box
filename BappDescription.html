DevSecBox is a versatile extension for orchestrating and automating security workflows in Burp Suite. It serves as a cross-platform command-line adapter, enabling execution of shell scripts, PowerShell, compiled programs, and other utilities. Through its visual interface, users can build complex workflows that combine security testing, system commands, and data transformation, integrating seamlessly with tools like Docker, Kubernetes, and nmap. With support for both live and manual modes, DevSecBox allows you to automate trigger-based logic, perform dynamic request/response processing, and generate custom outputsâ€”all within Burp Suite.

DevSecBox is ideal for DevSecOps teams, bug bounty hunters, and security engineers who need customizable, script-driven testing scenarios. From simple data filters to multi-step orchestration chains, it adapts to a wide range of use cases, including port scanning, mocking, container launching, infrastructure automation, and compliance-driven auditing.

Features

  • Visual workflow editor with drag-and-drop interface
  • Live and manual execution modes for flexible testing
  • Trigger-based automation for running scripts and programs
  • Request/response manipulation and spoofing
  • Built-in JSON formatting and conditional processing
  • Cross-platform command-line integration (bash, PowerShell, etc.)
  • Audit output support

Usage

  1. Start by designing a workflow using the drag-and-drop editor. Define a Trigger that captures data from the Proxy or Logger context menu.
  2. Right-click inside the DevSecBox panel to add a processing component (called a Catcher), which will handle and manipulate incoming data from the trigger.
  3. Write your logic directly in shell or PowerShell syntax. Use placeholder blocks which correspond to specific fields in the HTTP message.
  4. Add subsequent Catchers to transform the output into JSON or chain additional logic. Each Catcher receives data from the one before it.
  5. To test the setup manually, go to the Proxy menu, right-click a request, and choose "Send to DevSecBox". You can then view input and output data in each Catcher's "OUT" tab.
  6. Switch to Live mode for automated execution during browsing. Be aware that outputs may update too quickly to observe unless specific filters are used.
  7. Use the extended Live audit feature to populate structured issue reports by converting output into predefined JSON vulnerability formats.
